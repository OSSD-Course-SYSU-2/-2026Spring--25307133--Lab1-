实现一个简单的登录页面整体布局
使用Column容器垂直布局。在一行中排列多个元素使用Row容器进行水平布局。
页面中的组成元素包含图片（Image）、文本（Text）、输入框（TextInput）、按钮（Button）等。
完成了页面的布局分析后，接下来将登录页划分区域，并逐步实现。
登录页的区域可以划分为六部分。第一部分为登录页面的顶部logo图，第二部分为登录页面的文字描述，第三部分为账号密码输入部分，第四部分为“短信验证码登录”“忘记密码”部分，第五部分为登录按钮以及“注册账号”部分，第六部分为其他登录方式部分。
Index.ets是创建项目自动构建生成的，将其删除掉，并重新在page文件夹下创建一个LoginPage.ets文件，在其中完成登录页面的开发。
完成了LoginPage.ets文件的创建后，还需要对工程配置进行修改，以保证应用可以正常启动。
首先，需要在如右图所示的路径中的main_pages.json进行修改。

完成了main_pages.json文件的修改后，还需要在EntryAbility.ets文件中，设置应用启动时加载的页面。
需要修改onWindowStageCreate()方法中的windowStage.loadContent部分，设置其加载pages/LoginPage，使得应用启动后能够加载LoginPage。
至此就完成了基本的应用配置
完成了从Index到LoginPage的修改后，就可以开始结合之前的区域划分逐步实现了。首先，为LoginPage添加最基本的框架。
由于整体页面沿垂直方向布局，所以使用Column组件作为LoginPage的根容器组件。
接下来，为根容器组件Column设置宽高以及背景颜色等属性，宽高通过width属性和height属性设置，背景颜色则通过backgroundColor属性设置。
顶部logo通过使用Image组件实现。工程所需的图片资源可以从下载附件获取。首先，需要将图片资源放至media文件夹下。
将图片放至media目录后就可以通过$r的方式获取资源。通过向Image组件中传入$r获取到的Resource类型的对象，即可展示一张图片。
以右侧的代码为例，将下载附件中01_Resources文件夹中的logo.png复制到media文件夹下，之后就可以通过$r('app.media.logo')的方式，获取到该图片，并通过Image组件展示出来。
可以通过使用预览器，查看当前实现效果。
上面的预览图存在图片过大导致展示不全的问题，因此需要为图片设置宽高。
在登录页的目标效果中，logo图距离顶部，以及下方文字均有一定的间距，为此需要为logo设置一个顶部和底部的外边距，这需要使用margin属性来设置。
至此就完成了登录页第一区域，即顶部logo部分的开发了。
还需要修改文本的字体大小、粗细和颜色，分别可以通过fontSize、fontWeight和fontColor属性设置，并为第二个文字组件设置一定的上下外边距。
在实际开发中，为适配国际化，字符串需要抽取到string.json文件中做资源抽取，并通过$r的方式引用。在文档中，为了方便阅读，文字描述部分均直接使用中文字符串硬编码，源码中则做了资源的抽取.。
通过maxLength属性设置文本的最大输入字符数。
分别为账号密码设置11和8的字符数输入限制。
还需要通过type属性为账号密码输入框设置类型。
密码的类型设置为InputType.Password，账号设置为数字类型。
两个输入框的属性有很多都是重复的，并且属性值也一致。可以将这部分重复一致的属性使用@Extend提取出来。
最后，还需要为输入框底部增加一条下划线，使用Line组件，并设置宽度为100%，高度为1vp，并设置一个背景颜色。
同样的，将Line组件的重复属性使用@Extend做一个提取。
至此完成了登录页第三区域，即账号密码输入框部分的开发了。
可以看出，“短信验证码登录“及“忘记密码”部分，整体是沿水平方向布局，因此需要使用Row组件。
添加一个Row组件，并为其设置好width属性。
接下来向Row组件中添加两个文本组件Text，分别用于展示“短信验证码登录“与“忘记密码”。
目标效果需要使两个文本组件分别位于行首和行尾，当前效果与目标效果不一致。
因此需要对组成元素的排列进行一定的调整，主要使用的是布局容器Column和Row的justifyContent和alignItems属性，来设置布局容器组件在主轴和交叉轴上的对齐方式。
SpaceBetween会在主轴方向均匀分配元素，相邻元素之间距离相同。第一个元素与行首对齐，最后一个元素与行尾对齐。
在使用justifyContent设置第一个Row容器组件布局前为其添加width属性。
接下来修改两个文本组件的属性，包括字体大小、字体颜色等。
同样的，由于两个文本效果一致，所以依旧使用@Extend对这部分做一个提取。
最后，设置整个Row容器的外边距，使其与上方输入框有一定间距。
至此就完成了登录页第四区域，即“短信验证码登录“及“忘记密码”部分的开发了。
接下来，为按钮组件及文本组件设置属性，包括宽、高、字体大小、颜色等。
至此，我们就完成了登录页第五区域，即登录按钮及“注册账号”部分的开发了。
其他登录方式部分，主要由一个文本和一行按钮组成。
因此，首先补充一个框架，框架中包含一个文本组件Text，以及一个Row组件，并在Row组件中添加三个按钮组件Button。
接下来，为文本组件添加基本属性，包括字体颜色、字体大小、外边距等。
修改底部按钮的格式，由于在资源目录中提供了方式一、方式二、方式三的图片，所以使用Button直接展示这些图片，并为Button组件设置宽、高、背景颜色属性。
通过Button组件的参数ButtonOptions，可以设置按钮样式，以及按钮按下时是否开启按压态显示效果，由于图片本身为圆形，导致效果并不明显。
可以发现，方式一、方式二、方式三三个按钮，除了图片不同以外，其他的属性或参数配置，都是相同的。因此，使用@Builder自定义构建函数，进行这部分渲染的复用。封装了名为imageButton()的自定义构建函数，并将图片资源作为参数传入，其余的属性和参数不变，这样三个按钮仅需要调用该方法，并传入不同的图片资源，即可完成复用。

与目标效果对比，还需要为三个按钮间设置间距，使用Row组件的space参数，来设置横向布局元素间距。
在这里，设置space参数为44，从而保证三个按钮之间的间距为44。
至此，就完成了登录页第六区域，即其余登录方式部分的开发了。
最后，为最外层Column组件设置一个内边距，使得所有的子组件宽度上并不完全占满Column，与其左右两侧都保持一定的距离。这里使用padding属性设置内边距。
至此就完成了基础登录页面的开发了。
